Ready
System Architecture
Component breakdown and request flow from client to model API.
🌐
Chat Interfaces
Two frontends: a Next.js app (:3000) and a Gradio app (:7860). Both talk to the same FastAPI backend.
⚡
FastAPI Server (:8000)
Handles POST /invocations (MLflow), /v1/chat/completions (streaming proxy), and GET /health. Implements MLflow ResponsesAgent.
🧠
OpenAI Agents SDK
Manages the tool-calling loop, conversation context, and multi-step reasoning. System prompt is loaded from system_prompt.txt.
🔄
Fallback Client
Tries Nemotron → Qwen → GPT-4.1. All three tiers support tool calling. Raises RuntimeError when all are exhausted.
🔧
Tool Registry
Five tools dynamically loaded at startup via importlib: read_file, write_file, http_get, http_post, run_shell.
📊
MLflow Tracing
Every invocation is traced via mlflow.openai.autolog(). Spans export to Databricks workspace automatically.

Built-in Tools
The agent selects and calls these autonomously. All tools are in agents/perfect_agent/tools/.
read_file
read_file(path: str) → str
Read and return the full text contents of a file at the given path.
write_file
write_file(path: str, content: str) → str
Write or overwrite a file. The agent will request confirmation for destructive changes.
http_get
http_get(url: str) → str
Make an HTTP GET request and return the response body as text.
http_post
http_post(url: str, body: dict) → str
Make an HTTP POST request with a JSON payload and return the response.
run_shell
run_shell(cmd: str) → str
Execute a shell command and return combined stdout + stderr.
ℹ
Tool schemas are auto-generated from Python type annotations and registered with the OpenAI function-calling API. Every call appears in the MLflow trace.
agents/perfect_agent/runner.pyPython
# Tool resolution map
TOOLS = {
    "read_file":  "agents.perfect_agent.tools.file:read_file",
    "write_file": "agents.perfect_agent.tools.file:write_file",
    "http_get":   "agents.perfect_agent.tools.http:http_get",
    "http_post":  "agents.perfect_agent.tools.http:http_post",
    "run_shell":  "agents.perfect_agent.tools.shell:run",
}

def chat_with_agent(user_message: str) -> str:
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user",   "content": user_message},
    ]
    response = fallback_chat(messages, tools=tool_schemas())
    msg = response.choices[0].message
    if msg.tool_calls:
        results = [call_tool(c) for c in msg.tool_calls]
        final = fallback_chat(messages + results)
        return final.choices[0].message.content
    return msg.content

Setup & Deployment
Get the agent running locally in under five minutes, or deploy to Databricks Model Serving.
ℹ
Prerequisites: Python 3.11+, uv (pip install uv), and at least one of OPENROUTER_API_KEY or OPENAI_API_KEY.
1 — Quick start
TerminalShell
# Clone and install
git clone https://github.com/robert2687/agent-openai-agents-sdk
cd agent-openai-agents-sdk/agent-openai-agents-sdk
uv sync

# Configure credentials
cp .env.example .env
# Edit .env: set OPENROUTER_API_KEY and/or OPENAI_API_KEY

# Start API server + Gradio UI
uv run start-app
# API:    http://localhost:8000
# Gradio: http://localhost:7860
# Health: http://localhost:8000/health
2 — Full stack with Next.js UI
TerminalShell
uv run start-app --with-ui
# Or separately:
uv run start-server                           # API :8000
cd e2e-chatbot-app-next && npm i && npm run dev  # UI  :3000
3 — Databricks Model Serving
TerminalShell
databricks auth login
uv run quickstart   # registers + deploys to Model Serving
4 — Diagnostics
TerminalShell
curl http://localhost:8000/health
uv run verify-setup
uv run agent-evaluate
uv run discover-tools
